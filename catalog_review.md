# 既存カタログ素材レビューと改善提案

**対象素材**:
1. 事例集PDF（Case Studies.pdf）— 2ページ・三つ折りパンフレット
2. 会社紹介PDF（KiQ Robotics紹介資料.pdf）— 12ページ・プレゼンデッキ
3. 展示会ポスター（A0_tate.ai）— A0縦（1189×841mm）
4. 事例集Illustratorデータ（.ai × 2） — 表面・中面
5. 会社紹介PowerPoint（KiQ Robotic事業紹介.pptx）— 編集元データ

**レビュー方針**: マーケティング戦略で設定した「課題ベース・顧客視点」の方針に基づき、リード獲得・商談化に直結する改善点を優先度順に提示する

---

## 1. 事例集PDF（Case Studies.pdf）

### 現状評価

**良い点**:
- 実名企業（トヨタ自動車、リックス、ボッシュ、ロボットメーカー）の事例を掲載できている
- 「従来の問題点 → 導入後の効果」の構成は基本的に正しい
- 現場写真・イラストが豊富で、視覚的に理解しやすい
- QRコードで動画・記事への導線がある

**改善が必要な点**:

#### A. 情報量が薄い（最優先）

現状の「従来の問題点」が1〜2語のキーワードのみで、読み手の共感を得られない。

| 現状 | あるべき姿 |
|:---|:---|
| 「複数対象」 | 「数十種類の通い箱を扱うラインで、品種ごとにチャック交換が必要。段取り替えに1回30分、月に数十回発生」 |
| 「把持ミス」 | 「メカチャックではワーク形状の微妙な差異に対応できず、把持ミスが月○回発生。ライン停止による損失が課題」 |
| 「対象劣化」 | 「金属製メカチャックの接触でインジェクタ表面に傷がつき、不良品として廃棄。不良率○%が問題」 |
| 「爪劣化」 | 「ゲル製フィンガーが数万回で摩耗し、2〜3ヶ月ごとに交換が必要。交換コスト・ダウンタイムが課題」 |

#### B. 導入効果に定量データがない

「数十種類の把持」「400万回の耐久試験」は製品スペックであり、**顧客にとっての効果**ではない。

改善例:
- **トヨタ事例**: 「段取り替え: 月○回 → 0回（年間○時間の工数削減）」「チャック交換コスト: 年間○万円削減」「30件のリピート発注実績」
- **リックス事例**: 「把持ミス: 月○回 → 0回」「爪交換: ○ヶ月ごと → 100万回超で交換不要」
- **ボッシュ事例**: 「ワーク損傷による不良: ○% → ○%」「6件のリピート発注」

#### C. CTAが弱い

表紙にオンラインストアと把持試験サービスのQRコードがあるが、「次に何をすればいいか」が明確でない。

**改善案**:
```
■ まずは無料の把持テストをお試しください
1. ワークをお送りいただく（着払い可）
2. 弊社ラボで把持テストを実施（約1週間）
3. テスト結果とお見積もりをご報告
設計・見積もり・把持テスト すべて無料
```

#### D. 構成の改善

**現状の構成**（三つ折り）:
- 表面: 製品紹介 + 事例4
- 中面: 事例1〜3 + 原理説明

**推奨構成**:
- **表面（表紙）**: キャッチコピー（課題提起）+ 製品写真 + 「130件超の導入実績」
- **表面（裏表紙）**: CTA（無料把持テストの流れ）+ 問い合わせ情報
- **中面左**: こんな課題ありませんか？（4つの課題チェックリスト）
- **中面中央**: 事例1〜2（詳細版・定量データ付き）
- **中面右**: 事例3〜4 + 仕様概要

### 具体的な修正指示（Illustratorデータ向け）

1. 表紙のキャッチコピーを変更する
   - 現状: 「不定形・異形状対応が可能 / ツールチェンジャ・段取り替えレスへ」
   - 変更案: 「把持ミスゼロ。段取り替え不要。/ 1つのハンドで多品種ワークに対応」
2. 各事例の「従来の問題点」を具体的な文章に書き換える（上記表参照）
3. 各事例の「導入後の効果」に定量データを追加する（片山さんに実データ確認が必要）
4. 裏表紙に把持テストCTAを追加する
5. 「130件超の導入実績」「リピート率33%」を目立つ位置に入れる

## 2. 会社紹介PDF（KiQ Robotics紹介資料.pdf）

### 現状評価

**良い点**:
- 12ページで網羅的な内容（企業概要→事業→特長→導入メリット→事例6件→協業）
- J-Startup KYUSHU / KyuTech Startup認定のロゴが信頼性を補強している
- p5の「従来 vs 将来」比較が視覚的にわかりやすい
- 事例が6件と豊富（トヨタ、リックス、ボッシュ、デンソー、ロボットメーカー、東京大学）
- リックスとの共同開発パレデパレロボットの写真がインパクトある

**改善が必要な点**:

#### A. 「Confidential」表記の整理

全ページに「Confidential」が入っている。用途によって使い分けが必要:

| 用途 | Confidential | 対応 |
|:---|:---|:---|
| 投資家・提携先向けプレゼン | 必要 | 現状のまま |
| イプロス/メトリーのダウンロード資料 | 不要 | 外す |
| 営業配布用カタログ | 不要 | 外す |
| 展示会配布用 | 不要 | 外す |

→ **マーケティング用途では別バージョン（Confidential無し）を作成する**

#### B. 顧客の課題が見えない（最優先）

p3〜p4が「製品の説明」に終始しており、「なぜこの製品が必要か」が読み手に伝わらない。

**現状のp3**:
> 「ラティス構造柔軟指とは？ / ロボットの指先に取り付けるロボットハンドツールです」

**改善案**:
> 「こんな課題ありませんか？ / □ 把持ミスでラインが止まる / □ 多品種ワークの段取り替えに時間がかかる / □ ワークを傷つけてしまう / □ 爪・指の摩耗が早い」
>
> → この後に「柔軟指がすべて解決します」として製品紹介に入る

#### C. 事例の深みが不足（事例集PDFと同じ問題）

p6〜p10の事例が、事例集PDFと同様に「従来の問題点」「導入後の効果」がキーワードレベル。会社紹介資料では1ページ使えるので、以下の構成にする:

```
■ 事例タイトル（顧客名 + 用途）

【課題】（2〜3行で具体的に）
○○工程で△△の問題があり、□□のコストが発生していた。

【導入】（1〜2行）
柔軟指を○○に適用。カスタム設計で△△に対応。

【効果】（定量データ）
・把持ミス: 月○回 → 0回
・段取り替え: ○回/月 → 0回
・年間コスト削減: ○万円
・リピート: ○件の追加発注
```

#### D. 競合比較がない

生産技術者が社内稟議を通すときに必ず「他の選択肢との比較」が必要になる。以下を追加する:

```
         柔軟指       OnRobot SG   Robotiq 2F   メカチャック
価格     10〜30万円    約67万円      約45〜75万円   数万円〜
多品種   ◎            △           △            ×
耐久性   100万回+      要交換        ○           ◎
カスタム  ◎            ×           ×           △
```

#### E. 価格・納期情報がない

検討段階の顧客が最も知りたい情報が欠落している。「見積もり取らないと価格がわからない」状態は離脱の原因になる。

**追加する情報**:
- 標準品: 10万円〜（在庫あれば即納）
- カスタム設計: 15万円〜（設計・見積もり無料、納期約3週間）
- 把持テスト: 無料

#### F. CTAページがない

資料を見終わった後の行動導線がない。最終ページに以下を追加する:

```
■ まずは無料の把持テストから

1. お問い合わせ or ワーク送付
2. 弊社ラボでテスト（約1週間）
3. 結果報告 + お見積もり

すべて無料。お気軽にご相談ください。
```

### 推奨ページ構成（改訂版）

| ページ | 現状 | 改訂案 |
|:---:|:---|:---|
| 1 | 表紙 | 表紙（変更なし） |
| 2 | 企業概要 | 企業概要（変更なし） |
| 3 | 事業紹介（製品説明） | **こんな課題ありませんか？**（課題提起） |
| 4 | 製品特長 | **柔軟指とは** + 3つの強み（課題→解決の流れ） |
| 5 | 導入メリット（vs ツールチェンジャ） | 導入メリット（現状ベースでOK、コスト比較を追加） |
| 6 | 事例: トヨタ | 事例: トヨタ（定量データ追加） |
| 7 | 事例: リックス | 事例: リックス（定量データ追加） |
| 8 | 事例: ボッシュ | 事例: ボッシュ（定量データ追加） |
| 9 | 事例: デンソー | 事例: デンソー（定量データ追加） |
| 10 | 事例: ロボットメーカー | **競合比較表** + **仕様・価格・納期** |
| 11 | 事例: 東京大学 | 事例ダイジェスト（東京大学 + ロボットメーカー + その他を1ページに集約） |
| 12 | リックス共同開発 | **CTA: 無料把持テストのご案内** + 問い合わせ情報 |

→ 事例を絞って（代わりに深くして）、競合比較・価格・CTAページを確保する

## 3. 展示会ポスター（A0_tate.ai）

### 現状評価

**良い点**:
- ロボットアームに柔軟指を装着した大きなイラストが目を引く（展示会ブースでの視認性が高い）
- 把持の瞬間の実写真4枚で変形の様子を伝えている
- 「対象物の形状にならい しっかりキャッチ!!」のコピーがわかりやすい

**改善が必要な点**:

#### A. キャッチコピーが製品説明止まり

**現状**:
> 「ラティス構造柔軟指 / 不定形・異形状対応が可能 / ツールチェンジャ・段取り替えレスへ」

展示会で3秒以内に興味を引く必要がある。技術用語「ラティス構造」は通りすがりの来場者には伝わらない。

**改善案**:
> 「把持ミスゼロ。段取り替え不要。」（メイン・最大文字）
> 「1つのハンドで多品種ワークに対応するロボットハンドツール」（サブ）
> 「トヨタ・デンソー・ボッシュなど130件超の導入実績」（信頼性）

#### B. 導入実績・信頼性の訴求がない

展示会ポスターは「この会社は信頼できるか」を瞬時に判断される場。以下を目立つ位置に追加する:
- 「130件超の導入実績」
- 「リピート率33%」
- 導入企業ロゴ一覧（掲載許可があるもの）
- J-Startup KYUSHU / KyuTech Startup認定ロゴ

#### C. CTAがない

来場者がブースに立ち寄った後の行動導線が不明。以下を追加する:
- 「無料把持テスト実施中 — ワークをお送りください」
- QRコード → 問い合わせフォーム or オンラインストア（現状のQRが何に遷移するか要確認）

#### D. 仕様・差別化情報がない

展示会では競合ブースと比較される。最低限以下を入れる:
- 「OnRobot / Robotiq互換」（既存設備に導入可能）
- 「100万回以上の耐久性」（ゲル・スポンジの数十倍）
- 「10万円〜」（競合ソフトグリッパーの1/3以下）
- 「カスタム設計無料」

#### E. 印刷テンプレートの注記が残っている

下部に「再入稿にならないためのポイントはこちら」というピンクのバナーが入っている。印刷会社のテンプレート注記なので、入稿時に削除が必要（単なる注意事項なら問題ないが、印刷物に残っていないか確認する）。

## 4. 素材間の整合性

### 名称の統一

| 箇所 | 現状 | 統一案 |
|:---|:---|:---|
| 事例集PDF表紙 | 「ラティス構造柔軟指」 | ロボットハンドツール「柔軟指」 |
| 会社紹介p3 | 「ラティス構造柔軟指」 | ロボットハンドツール「柔軟指」 |
| ポスター | 「ラティス構造柔軟指」 | ロボットハンドツール「柔軟指」 |
| イプロス改修案 | ロボットハンドツール「柔軟指」 | — |

→ 検索キーワードを意識して「ロボットハンドツール」を必ず冠する。「ラティス構造」は技術説明内で使う

### 数値の統一

| 項目 | 事例集PDF | 会社紹介PDF | イプロス改修案 | 統一値 |
|:---|:---|:---|:---|:---|
| 耐久性 | 400万回 | 400万回 | 100万回以上（450万回の実績） | 「100万回以上（最大450万回の実績）」に統一 |
| 導入実績 | 記載なし | 記載なし | 130件超 | 全素材に「130件超」を明記 |
| リピート率 | 記載なし | 記載なし | 33% | 全素材に追加 |
| 互換性 | OnRobot, Robotiq, SMC | 記載なし | OnRobot RG2/RG6, Robotiq 2F-85/2F-140/Hand-E | 具体型番を全素材に統一 |
| 対応ハンド | SMC記載あり | SMC記載なし | SMC記載なし | SMC対応が現在もあるなら追加する。なければ削除する |

### メッセージの統一

全素材で一貫させるべきコアメッセージ:

1. **把持ミスゼロ・段取り替え不要** — 顧客課題の直接解決
2. **130件超の導入実績、リピート率33%** — 信頼性の証明
3. **競合の1/3以下の価格** — 経済合理性
4. **設計・見積もり・把持テスト無料** — 行動障壁の除去

## 5. 全素材の課題サマリー

| 素材 | 最優先の課題 | 改善インパクト |
|:---|:---|:---|
| **事例集PDF** | 事例の情報が薄い（キーワードのみ、定量データなし） | 営業・展示会での説得力が大幅向上 |
| **会社紹介PDF** | Confidentialで配布不可 + 課題提起・競合比較・価格・CTAが欠落 | イプロス/メトリーのダウンロード資料として使用可能に |
| **ポスター** | キャッチコピーが技術用語止まり + 実績・CTA・差別化情報なし | 展示会でのブース集客力が向上 |
| **全素材共通** | 名称・数値がバラバラ + コアメッセージが不統一 | ブランドの一貫性確保 |

## 6. 優先度順アクションリスト

### Phase 1: すぐできること（1〜2週間）

1. **名称・数値の統一表を確定する** — 上記のセクション4の統一案をチームで確認し、全素材に適用する基準を決める
2. **事例の定量データを社内で収集する** — 各事例の「段取り替え削減回数」「把持ミス削減回数」「コスト削減額」「リピート件数」を営業担当に確認する
3. **会社紹介PDFのConfidential無しバージョンを作成する** — pptxから編集、マーケティング配布用
4. **会社紹介PDFに以下を追加する**（pptx編集）:
   - p3を課題提起ページに変更する
   - p10に競合比較表を追加する
   - 最終ページにCTA（無料把持テスト案内）を追加する
   - 価格帯・納期を追加する
5. **ポスターのキャッチコピー・情報を改訂する** — Illustratorデータを編集し、課題ベースのコピー・実績・CTA・差別化情報を追加する

### Phase 2: 定量データ収集後（2〜4週間）

6. **事例集PDFの全面改訂** — Illustratorデータを使って、事例を定量データ付きに書き換える
7. **イプロス用カタログPDF（catalog_pdf_draft.md）をCanvaで制作する** — 事例集と会社紹介の素材を流用する
8. **メトリー用にも同一PDFを登録する**

### Phase 3: 新規制作（1〜2ヶ月）

9. **課題解決ガイドPDF**（catalog_pdf_draft.mdのPDF 2）を制作する — リードナーチャリング用

## 7. 補足: 事例掲載の許可確認

会社紹介PDF（Confidential）では企業名を出しているが、マーケティング用途（イプロス・配布資料）で使う場合は掲載許可の再確認が必要:

| 企業 | 現在の掲載状況 | マーケ用途での使用可否 |
|:---|:---|:---|
| トヨタ自動車 | 会社紹介PDF + 事例集PDF + YouTube | 要確認 |
| リックス | 会社紹介PDF + 事例集PDF | 要確認（資本関係あり、許可を得やすい可能性が高い） |
| ボッシュ | 会社紹介PDF + 事例集PDF | 要確認 |
| デンソー | 会社紹介PDF | 要確認 |
| 東京大学 | 会社紹介PDF | 要確認 |
| ロボットメーカー（社名非公開） | 会社紹介PDF + 事例集PDF | そのまま使用可能 |

→ 許可が取れない企業は「大手自動車メーカーA社」等の匿名表記にする
