# 第1週 動画制作ガイド（2026年3/31〜4/4）

**目的**: バズ分析の知見を反映した「冒頭3秒フック」「畏敬感情」「完視聴率70%超」を意識した制作仕様書

---

## 共通の機材・セット設定

| 項目 | 内容 |
|:---|:---|
| **カメラ** | スマホ縦型（9:16）。三脚固定が基本 |
| **背景** | グレー or 薄紫の無地クロス。手持ちのものでOK |
| **照明** | 正面上45°から1灯＋レフ板（白い紙でも可）。影が濃すぎないように |
| **解像度** | 1080×1920以上 |
| **フレームレート** | 60fps で撮影（スロー編集できるように） |
| **テロップフォント** | 太めのゴシック体（Noto Sans Black等）。白文字＋黒縁取り |

## 月曜（3/31）— フォーマットA「衝撃デモ型」

**タイトル案**: 絹ごし豆腐、ロボットで掴んだらどうなる？
**目標尺**: 20〜22秒
**狙い**: 「壊れる予感」→「壊れない驚き」で畏敬感情を誘発。TikTok完視聴率70%超

### 用意するもの

- 絹ごし豆腐 × 3パック（撮り直し用）
- 白い小皿 or 薄いグレーのトレー
- 柔軟指（把持力を一番弱い設定に）

---

### カット割り・アングル・テロップ

#### カット1（0〜2秒）— フック

| 項目 | 内容 |
|:---|:---|
| **アングル** | 真上から真下に向けた俯瞰（トップダウン）。豆腐がド真ん中に収まるように |
| **映像** | 皿の上の絹ごし豆腐のみ。静止画に近いくらいゆっくりした状態 |
| **テロップ** | 画面上部に大きく「絹ごし豆腐」、その下に小さく「壊さずに掴めると思う？」 |
| **フォントサイズ** | 「絹ごし豆腐」80pt、サブテキスト40pt |
| **BGM** | まだ入れない。無音で緊張感を出す |

> **狙い**: 1秒目で「え、豆腐？」と思わせて止める。テロップが問いかけになっていることが重要

---

#### カット2（2〜8秒）— 失敗演出（期待値操作）

| 項目 | 内容 |
|:---|:---|
| **アングル** | 斜め45°の正面やや上から。柔軟指と豆腐が両方フレームに入る高さ |
| **映像** | 柔軟指がゆっくり豆腐に近づく。接触の瞬間に**一時停止2秒**。視聴者に「割れるかも」と思わせる |
| **テロップ** | 画面下部に「果たして…」（小さめ、40pt） |
| **BGM** | 緊張感のある低音SE（ドラムロールや心臓音など1〜2秒）を接触直前に入れる |
| **撮影メモ** | 接触瞬間は60fpsで撮ってスロー編集（0.5倍速）にすると豆腐の微妙な変形が見える |

> **狙い**: 「割れる？割れない？」の緊張が視聴継続を促す。焦らすほど完視聴率が上がる

---

#### カット3（8〜16秒）— 成功の瞬間（オチ）

| 項目 | 内容 |
|:---|:---|
| **アングル** | 斜め45°正面。柔軟指が豆腐を持ち上げたところで**横から寄りのカット**に切り替え |
| **映像** | 豆腐を10cmほど持ち上げ→3秒静止（形が崩れていないことを見せる）→ゆっくり戻す |
| **テロップ1** | 持ち上げた瞬間に画面中央「SUCCESS!!」（赤文字、黒縁、80pt）を0.5秒で出す |
| **テロップ2** | その後「形を保ったまま把持」（白文字、50pt）に切り替え |
| **BGM** | 緊張BGMが止まり→軽快なポップSE（「ポン！」系）→明るいBGMに切り替え |
| **撮影メモ** | 豆腐を戻すときの「離す瞬間」も60fpsで撮る。崩れていないことが一番のクライマックス |

---

#### カット4（16〜20秒）— ブランド提示

| 項目 | 内容 |
|:---|:---|
| **アングル** | 少し引いて柔軟指全体とKiQ Roboticsロゴが見える構図 |
| **映像** | 柔軟指をゆっくり回転させる or 静止 |
| **テロップ** | 「柔軟指 / Lattice Hand」（白、60pt）＋「KiQ Robotics」（グレー、30pt） |
| **BGM** | 明るいBGMが継続 |

> **注意**: KiQ Roboticsのロゴが**逆向きにならないよう**、カメラに向けて持つ方向を確認する

---

### 編集チェックリスト（月曜）

```
□ カット1の俯瞰で豆腐がど真ん中に収まっているか
□ カット2のスロー再生で変形が見えるか
□ 接触直前のSEが入っているか
□ 「SUCCESS!!」テロップが持ち上げた瞬間に出ているか
□ KiQ Roboticsロゴが正しい向きか
□ 全体尺が25秒以内か
□ 無音でテロップだけでも内容が伝わるか（無音視聴テスト）
```

## 火曜（4/1）— フォーマットC「比較対決型」

**タイトル案**: りんごとみかん、同じ指で掴い分けできる？
**目標尺**: 25〜30秒
**狙い**: サイズ・形が違うものを1本の指で対応できることを視覚で証明。「金型不要」訴求

### 用意するもの

- りんご × 2個（同サイズ）
- みかん × 2個
- Normal Hand（対照用）
- 柔軟指
- グレーのトレー or 木目テーブル背景

---

### カット割り・アングル・テロップ

#### カット1（0〜3秒）— フック

| 項目 | 内容 |
|:---|:---|
| **アングル** | 正面から。りんごとみかんを左右に並べて置く |
| **映像** | りんご（左）・みかん（右）が画面に収まった状態。手がフレームイン |
| **テロップ** | 中央上部「りんごとみかん」（白80pt）、下部「同じ指で掴い分けできる？」（黄色50pt） |
| **BGM** | 軽快なBGMを最初から。テンポ感があるもの |

---

#### カット2（3〜12秒）— Normal Hand パート

| 項目 | 内容 |
|:---|:---|
| **アングル** | 画面**左半分**にNormal Handを配置。右半分は空白（後でLattice Hand用） |
| **映像** | Normal Handでりんごを掴む→成功→次にみかんに持ち替えようとする→爪が合わず失敗 or 落とす |
| **テロップ1** | 左上に「Normal Hand」（白、太字） |
| **テロップ2** | りんご掴んだ時「りんご: OK ✓」→みかん失敗時「みかん: NG ✗」（赤文字） |
| **SE** | 失敗時に「ブッ」と短いネガティブSE |
| **撮影メモ** | Normal Handでみかんを掴もうとして「爪が合わない」シーンをしっかり撮る。理由が視覚でわかること |

---

#### カット3（12〜22秒）— Lattice Hand パート

| 項目 | 内容 |
|:---|:---|
| **アングル** | 画面**右半分**にLattice Handを配置。または切り替えで全画面に |
| **映像** | りんごをスムーズに掴む→離す→みかんに切り替え→同じようにスムーズに掴む |
| **テロップ1** | 「Lattice Hand」（白、太字） |
| **テロップ2** | りんご「OK ✓」→みかん「OK ✓」（緑文字） |
| **テロップ3** | 両方成功後「金型交換なし」（黄色大文字、1秒表示） |
| **SE** | 各成功時に「ポン！」の短い効果音 |
| **撮影メモ** | りんごとみかんの切り替えは連続で。テンポよくスムーズに見せる |

---

#### カット4（22〜28秒）— まとめテロップ

| 項目 | 内容 |
|:---|:---|
| **アングル** | 引いて全体を映す |
| **映像** | りんごとみかんを両方持った状態（2本の柔軟指で）か、並べて置いた状態 |
| **テロップ** | 「1本の指で多品種対応」（白、大文字）＋「段取り替えゼロ」（黄色） |
| **BGM** | 明るく締まるBGM |

---

### 編集チェックリスト（火曜）

```
□ フックで「りんごとみかん両方映っているか」
□ Normal Handの失敗が視覚的にわかるか
□ 「りんご: NG ✗」のテロップが赤で見えるか
□ Lattice Handの切り替えがスムーズか
□ 「金型交換なし」テロップが入っているか
□ 無音テスト：テロップだけで「対決→結果」が伝わるか
```

## 水曜（4/2）— フォーマットD「課題解決型」

**タイトル案**: 【把持ミス対策】精密コネクタを安定把持してみた
**目標尺**: 35〜45秒
**狙い**: 製造業の「把持ミス」課題に刺さるSEOコンテンツ。終盤でCTA

### 用意するもの

- 小型コネクタ × 数個（できれば種類違いを2〜3種）
- 柔軟指（通常設定）
- テーブルor専用台

---

### カット割り・アングル・テロップ

#### カット1（0〜4秒）— 課題提起フック

| 項目 | 内容 |
|:---|:---|
| **アングル** | 黒背景でコネクタのクローズアップ（マクロ気味に）。精密さが伝わる構図 |
| **映像** | コネクタが静止した状態。動かない |
| **テロップ** | 上段「精密部品の把持ミスで」（白、55pt）→下段「ライン停止していませんか？」（赤、55pt） |
| **BGM** | 重めの低音BGMで始まる。工場っぽいアンビエント感 |

> **狙い**: 製造業の担当者が「これは自分の話だ」と感じる課題ワードから入る

---

#### カット2（4〜18秒）— 把持デモ（安定性の証明）

| 項目 | 内容 |
|:---|:---|
| **アングル** | 斜め45°正面。コネクタ全体と柔軟指が両方見える |

| **テロップ1** | カウンター表示「1回目」「2回目」…「5回目」（右下に小さく） |

| **撮影メモ** | 5回連続を一気に撮る。1回でも落としたら撮り直し。スピード感が大事 |

---

#### カット3（18〜30秒）— 多種類対応デモ

| 項目 | 内容 |

| **アングル** | 少し引いて3種類のコネクタが並んでいる構図 |

| **テロップ3** | 「金型交換ゼロ」（黄色、太字） |
| **SE** | 各把持成功時に「ポン」と短いSE |

---

#### カット4（30〜42秒）— CTA

| 項目 | 内容 |
|:---|:---|
| **アングル** | 柔軟指全体が見えるミドルショット |
| **映像** | 柔軟指を静止させたまま。製品のビジュアルを見せる |

## 木曜（4/3）— フォーマットA「衝撃デモ型（精密系）」

**タイトル案**: 柔軟指でトランプタワーに挑戦してみた
**目標尺**: 25〜35秒
**狙い**: 「崩れそう」というハラハラ感→「成功」の落差で畏敬感情。失敗シーンも使う

### 用意するもの

- トランプ × 1デッキ
- 平らなテーブル（揺れないもの）
- 柔軟指（把持力を弱く設定）

---

### カット割り・アングル・テロップ

#### カット1（0〜3秒）— フック

| 項目 | 内容 |
|:---|:---|
| **アングル** | 正面やや低め（タワーが高く見える構図）。3〜4段のトランプタワー |
| **映像** | タワーが静止。そこへ柔軟指がゆっくりフレームイン |
| **テロップ** | 「トランプタワーを柔軟指で触れてみた」（白、60pt） |
| **BGM** | 静かな緊張BGM（弦楽器のゆっくりしたもの） |

---

#### カット2（3〜15秒）— 失敗シーン（あえて使う）

| 項目 | 内容 |
|:---|:---|
| **アングル** | 斜め45°。柔軟指がタワーに近づく様子が見える |
| **映像** | 柔軟指がタワーに触れる→タワーが崩れる（1回目の失敗） |
| **テロップ** | 崩れる瞬間「あ…」（大文字、白、笑いを誘うフォント）→「1回目 失敗」（赤、小さめ） |
| **SE** | カード崩れる音（リアル音）→「ガシャーン」系SE |
| **BGM** | 崩れた瞬間にBGM止まる |
| **撮影メモ** | 失敗は意図的に1回撮っておく。視聴者が「あー！」と反応できる場面になる |

> **狙い**: 失敗→再挑戦の構造がドラマになり、最後まで見たくなる。コメントも「惜しい！」「次は？」が増える

---

#### カット3（15〜27秒）— 成功シーン

| 項目 | 内容 |
|:---|:---|
| **アングル** | 斜め45°正面。タワーが画面中央。柔軟指が上からゆっくり降りてくる |
| **映像** | 柔軟指がタワーの1枚に静かに触れる→タワーが揺れる（スロー）→崩れない→そのまま持ち上げる |
| **テロップ1** | 持ち上げ成功の瞬間「成功！！」（金色 or 白、大文字、アニメーション） |
| **テロップ2** | 「繊細なコントロールが可能」（白、50pt） |
| **SE** | 静寂の後「ファンファーレ」短め or 歓声SE |
| **BGM** | 成功と同時に明るいBGMに転換 |
| **撮影メモ** | 60fpsで撮影してスロー編集。タワーが微妙に揺れている様子が「ギリギリ感」を演出する |

---

#### カット4（27〜33秒）— ブランド締め

| 項目 | 内容 |
|:---|:---|
| **映像** | 成功したトランプタワーと柔軟指が一緒に映る状態で静止 |
| **テロップ** | 「柔軟指 / Lattice Hand」（白）＋「把持力を自由にコントロール」（グレー） |
| **BGM** | 落ち着く |

---

### 編集チェックリスト（木曜）

```
□ フックでタワーの高さと柔軟指の大きさの対比が見えるか
□ 失敗シーン「あ…」テロップが笑いを誘えるか
□ 成功シーンがスローモーションになっているか
□ 成功の瞬間BGMが転換しているか
□ 全体でドラマ（失敗→再挑戦→成功）の構造になっているか
□ 尺が35秒以内か
```
