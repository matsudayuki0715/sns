## 表記規則
- **文末**: 箇条書きは指示文（〜する）で統一する。末尾の「。」は付けない
- **ラベル**: `**ラベル名**: 説明文` の形式で統一する
- **句読点**: 全角「、」「。」を使用する。半角カンマ・ピリオドは英文・コード内のみ使用する
- **強調語**: 「IMPORTANT」「必ず」は本当に致命的な箇所のみに限定する（2〜3箇所まで）

## 意思決定の原則
- **品質優先**: 速度より品質を重視する
- **文献ベース**: 戦略設計・評価手法・パラメータ選定など、あらゆる判断で文献の裏付けを取る。指示がなくても常にこのルールに従う
- **調査ソース（学術）**: Google Scholar, ResearchGate, arXiv, SSRN, IEEE, ACM等。英語主軸、2020年以降優先
- **文献記録**: 著者・年・主要知見を記録し、`reports/research_algo_evaluation_metrics.md` に追記する
- **実装ベース**: ライブラリ選定・設計パターン・実装手法など、あらゆる実装判断で実際のコードや事例を十分に調査する。指示がなくても常にこのルールに従う
- **調査ソース（実装）**: GitHub（リポジトリ・Issues・Discussions）, Qiita, Zenn, Stack Overflow, Dev.to, 各公式ドキュメント等。スター数・更新頻度・メンテナンス状況を考慮する
- 調査後は情報充足性を評価し、不足・根拠が弱い箇所は追加調査ループを回す

## ファイル管理
- `tmp/` フォルダは常に空に保つ。一時ファイルは確認後すぐに適切な場所へ移動するか削除する

## Git
- 区切りのよいタイミングでコミット・プッシュする（ユーザーの確認不要）
- `gh` CLIを使ってリポジトリ作成・プッシュする。`git` の直接認証設定は行わない
- ユーザー設定は `gh api user` でアカウント名を取得し、以下の形式でリポジトリローカルに設定する:
  - `git config user.name "<GitHubアカウント名>"`
  - `git config user.email "<GitHubアカウント名>@users.noreply.github.com"`
- グローバル設定（`--global`）は変更しない
